# Smart Pharmacy Analytics

## Project Overview

Smart Pharmacy Analytics is an end-to-end data analytics project that demonstrates how raw pharmacy data can be transformed into meaningful business insights using Python, SQL, and Power BI.

The project focuses on analyzing prescription, patient, supplier, and medicine data to understand business performance, identify trends, and support data-driven decision-making.



## Problem Statement

Pharmacy businesses handle large amounts of operational data every day. Without proper analysis, it becomes difficult to monitor revenue, identify high-performing medicines, understand patient behavior, evaluate supplier performance, and improve overall business efficiency.

The goal of this project is to build a complete analytics solution that converts raw data into actionable business insights.



## Dataset

The project uses publicly available pharmacy-related datasets containing information about:

- Patients
- Doctors
- Medicines
- Suppliers
- Insurance
- Prescriptions

The datasets were merged and cleaned using Python to create a single analytical dataset.

Additional business metrics such as Revenue, Cost, and Profit were created for analysis.

To demonstrate the complete analytics workflow, a representative dataset of **15 prescription records** was prepared and used throughout the project.

The dataset is included in this repository.

## Key Business Insights

- Generated **8.97K** in total revenue with an overall profit of **1.41K** and a **15.75%** profit margin.
- Identified **Neurontin** as the highest revenue-generating medicine.
- Found that **McKesson** contributed approximately **52.7%** of total revenue, highlighting supplier dependency.
- Analyzed **6 unique patients** across **15 prescriptions**.
- Observed that most prescriptions had either **0** or **5** refills, indicating both short-term and recurring medication patterns.
